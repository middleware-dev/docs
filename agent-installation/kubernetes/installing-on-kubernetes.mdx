**Use following command to install agent on Kubernetes**
```javascript Shell
MW_API_KEY=xxxxxxxxxxxxxxxxxx TARGET=https://uid.middleware.io:443 bash -c "$(curl -L https://install.middleware.io/scripts/yaml-install.sh)"
```
<p>This script will install a Kubernetes daemonset into your k8s cluster.</p>
<p>This agent is mainly created to monitor the data of your Kubernetes infrastucture.</p>
<Info> Replace the MW_API_KEY value with your API key or directly copy the full command with the API key from the <a href="https://app.middleware.io/auth/login" rel="nofollow noopener noreferrer" target="_blank" translate="no">installation menu.</a></Info>
<Info> Replace "uid" with the text in the URL (ex URL is "s05zpimz.middleware.io" , here s05zpimz is uid.)</Info>

<Note>You must not run more than one MW Agent per cluster. Running multiple Agents may result in unexpected behavior.</Note>

## Verify Status for K8s Infrastructure Agent
<p>Run this command to check the status of main daemonset component, which will be collecting data from your cluster</p>

```bash
kubectl get daemonset/mw-kube-agent -n mw-agent-ns
```

<p>Expected Output:</p>

<table><tbody><tr><td >Name</td><td >Desired</td><td>Current</td><td>Ready</td><td >Up-to-date</td><td>Available&nbsp;</td><td>Non selector</td><td>Age</td></tr><tr><td>mw-kube-agent</td><td>7</td><td>7</td><td>7</td><td>7</td><td>7</td><td>&lt;none&gt;</td><td>111s</td></tr></tbody></table>

<p>If you have "N" number of k8s nodes, You should see "N" ready pods.</p>
